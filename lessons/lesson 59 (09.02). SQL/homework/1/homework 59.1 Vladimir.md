# Домашнее задание 59.1: Операции с базами данных SQL

## Описание задачи

Создайте структуру базы данных для хранения информации о книгах и авторах, используя только запросы для создания таблиц и удаления данных. После удаления данных таблицы должны сохраниться. Все записи должны быть добавлены и удалены с помощью SQL-запросов без удаления самих таблиц.

## Инструкции

### 1. Настройка базы данных
Создайте базу данных с именем `BookStore` (в SQLite база данных будет создаваться автоматически при подключении).

### 2. Создание таблицы Authors
Создайте таблицу `Authors` со следующими полями:
- `AuthorID` - уникальный идентификатор автора (тип INTEGER)
- `FirstName` - имя автора (тип TEXT)
- `LastName` - фамилия автора (тип TEXT)

### 3. Создание таблицы Books
Создайте таблицу `Books` со следующими полями:
- `BookID` - уникальный идентификатор книги (тип INTEGER)
- `Title` - название книги (тип TEXT)
- `AuthorID` - идентификатор автора, ссылающийся на таблицу Authors (тип INTEGER)
- `Price` - цена книги (тип REAL)

### 4. Очистка данных в таблицах
Удалите все записи из таблиц, не удаляя сами таблицы. Используйте для этого команду DELETE.

### 5. Сохранение структуры таблиц
Сохраните таблицы до и после удаления данных. Структура таблиц должна остаться в базе данных после выполнения запросов на удаление.

## Шаги выполнения

### Шаг 1: Создание таблицы Authors
Используйте команду создания таблицы с:
- AuthorID как первичный ключ
- FirstName и LastName как текстовые поля

### Шаг 2: Создание таблицы Books
Используйте команду создания таблицы с:
- BookID как первичный ключ
- Title как текстовое поле
- AuthorID как целочисленное поле (для связи по внешнему ключу)
- Price как поле с вещественным числом

### Шаг 3: Очистка всех данных
Выполните команды удаления для удаления всех записей:
- Сначала удалите из таблицы Books (для соблюдения ограничений внешнего ключа)
- Затем удалите из таблицы Authors

### Шаг 4: Проверка структуры таблиц
Убедитесь, что:
- Таблицы существуют в базе данных
- Таблицы пусты (нет записей)
- Структуры таблиц (колонки, типы данных) сохранены

## Ожидаемый результат

- CSV-экспорты таблиц Authors и Books в двух версиях: до и после выполнения DELETE-запросов
- Все SQL-запросы, использованные в решении

## Критерии оценивания

- **4 балла** - Создание таблиц Authors и Books
- **4 балла** - Очистка данных в таблицах с помощью команды DELETE без удаления самих таблиц
- **4 балла** - Таблицы сохранены с их структурой после удаления данных

**Максимальный балл за задание - 12 баллов**

## Подход к решению (псевдокод)

```
// Шаг 1: Инициализация подключения к базе данных
ПОДКЛЮЧИТЬСЯ к базе данных с именем 'BookStore'

// Шаг 2: Создание структуры таблицы Authors
СОЗДАТЬ ТАБЛИЦУ Authors С:
    - AuthorID (целое число, первичный ключ)
    - FirstName (текст)
    - LastName (текст)

// Шаг 3: Создание структуры таблицы Books
СОЗДАТЬ ТАБЛИЦУ Books С:
    - BookID (целое число, первичный ключ)
    - Title (текст)
    - AuthorID (целое число, ссылка на Authors)
    - Price (вещественное число)

// Шаг 4: Очистка всех данных из таблиц
УДАЛИТЬ все записи ИЗ Books
УДАЛИТЬ все записи ИЗ Authors

// Шаг 5: Проверка результатов
ЭКСПОРТИРОВАТЬ структуры таблиц для подтверждения их существования
ЭКСПОРТИРОВАТЬ содержимое таблиц для подтверждения их пустоты
```

## Важные замечания

- В SQLite файл базы данных создается автоматически при подключении
- Всегда удаляйте сначала из дочерних таблиц (Books), затем из родительских (Authors)
- DELETE удаляет данные, но сохраняет структуру таблицы
- DROP TABLE удаляет всю таблицу полностью - не используйте эту команду
