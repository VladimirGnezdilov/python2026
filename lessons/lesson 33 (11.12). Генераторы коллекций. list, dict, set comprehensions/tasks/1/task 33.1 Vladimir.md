# Задание 33.1. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Введение в генераторы коллекций» — отрабатываем умение увидеть. где выражение comprehension заменяет шаблонный цикл и делает код прозрачнее.

## Условие задачи
**Сложность**: <span style="color: #c62828">● сложный</span>. примерное время: 15–20 минут

Тебе нужно собрать отчёт о замерах температуры сервера. Исходные данные — список словарей с ключами `"time"`, `"cpu"`, `"gpu"`. Сделай одним list comprehension список строк `"time → max_temp"`, где `max_temp` — максимальное значение между `cpu` и `gpu`, но только если хотя бы один из показателей превышает 75°. Остальные записи пропусти.

## Пример входных данных
```
metrics = [
    {"time": "10:00", "cpu": 68, "gpu": 70},
    {"time": "10:05", "cpu": 81, "gpu": 74},
    {"time": "10:10", "cpu": 77, "gpu": 79}
]
```

## Пример выходных данных
```
['10:05 → 81°C', '10:10 → 79°C']
```

## Критерии успешного выполнения
- Вся логика реализована одним comprehension без внешних циклов.
- Используется `max` для выбора температуры между `cpu` и `gpu`.
- В результат попадают только записи с перегревом > 75°.
- Формат строки соблюдён и включает символ градусов.

## Подсказки
- Объедини фильтр и преобразование: `[f"{m['time']} → {max(m['cpu'], m['gpu'])}°C" for m in metrics if ...]`.
- Условие перегрева можно записать как `if max(...) > 75`.

## Частые ошибки и ограничения
- Не вычисляй `max` дважды. лучше сохранить в переменную внутри выражения через моржовый оператор `:=`, если используешь Python 3.8+.
- Не изменяй словари в исходном списке.
- Избегай добавления сложной бизнес-логики внутрь comprehension.
