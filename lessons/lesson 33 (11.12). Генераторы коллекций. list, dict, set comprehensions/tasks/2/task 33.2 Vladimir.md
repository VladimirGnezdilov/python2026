# Задание 33.2. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«List comprehensions» тренирует умение собирать данные из разных источников и фильтровать их в одном выражении.

## Условие задачи
**Сложность**: <span style="color: #c62828">● сложный</span>. примерное время: 15–20 минут

Есть список `events`, где каждый элемент — словарь с ключами `"title"`, `"slots"`, `"tags"`. Поле `"tags"` содержит список строк. Построй одним list comprehension список строк формата `"<title> — <count> подходящих тегов"` только для событий, где хотя бы два тега из набора `target_tags` встречаются в их `"tags"`, а доступных слотов (`"slots"`) больше нуля. В строке `<count>` — количество совпавших тегов.

## Пример входных данных
```
events = [
    {"title": "WebConf", "slots": 5, "tags": ["web", "python", "ml"]},
    {"title": "DataCamp", "slots": 0, "tags": ["data", "ml"]},
    {"title": "AsyncMeetup", "slots": 8, "tags": ["python", "async", "devops"]}
]
target_tags = {"python", "async", "data"}
```

## Пример выходных данных
```
['WebConf — 1 подходящих тегов', 'AsyncMeetup — 2 подходящих тегов']
```

## Критерии успешного выполнения
- Используется одно выражение list comprehension без промежуточных циклов.
- В результат попадают только события с `slots > 0` и минимум двумя совпадениями тегов.
- Подсчёт совпадений происходит внутри comprehension.
- Формат строки точно совпадает с образцом.

## Подсказки
- Внутри comprehension вычисли пересечение через `len(target_tags & set(event["tags"]))`.
- Фильтр по количеству совпадений поставь в конце выражения.
- Не забудь, что `target_tags` — множество, с ним удобно считать пересечения.

## Частые ошибки и ограничения
- Создание отдельных циклов и `append` нарушает требование.
- Нельзя модифицировать исходный список событий.
- Следи за читаемостью: одно выражение — одна идея.
