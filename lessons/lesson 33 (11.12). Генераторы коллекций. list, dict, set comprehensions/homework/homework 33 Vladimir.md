# Домашнее задание. Урок 33. Генераторы коллекций. list, dict, set comprehensions

## Краткая сводка
- По задачам урока 33 решений от Владимира нет. поэтому необходимо закрыть все основные блоки: list, dict и set comprehensions.
- Фокус на отработку фильтрации в одном выражении, аккуратного форматирования строк и расчета агрегатов без дополнительных циклов.

## Задача 1. Повторение темы. «Температурные значения серверов»
- **Блок:** list comprehension.
- **Описание:** дан список словарей `metrics = [{"time": "10:00", "cpu": 68, "gpu": 70}, ...]`. Сформируй список строк `"10:05 -> 81°C"` только если максимум между `cpu` и `gpu` превышает 75. Нумерацию менять не нужно.
- **Критерии:** одно выражение list comprehension, внутри используется `max`, условие фильтрации встроено в comprehension, исходный список не изменяется.

## Задача 2. Закрепление практики. «Платежи клиентов»
- **Блок:** list comprehension с форматированием.
- **Описание:** есть список `payments = [("Ирина", 32000, "Казань"), ("Марк", 58000, "Москва"), ...]`. Получи строки `"Клиент: Имя, сумма: VALUE₽"` только для городов на букву `К` и сумм ≥ 30 000. Для сумм ≥ 60 000 выводи `VALUEK`.
- **Критерии:** один list comprehension, корректное добавление символа валюты или суффикса `K`, исходный список не изменяется, нет промежуточных переменных.

## Задача 3. Интеграция блоков. «Режим работы отделов»
- **Блок:** dict comprehension.
- **Описание:** список `departments = [{"name": "Core", "hours": 28, "remote": True}, ...]`. Нужно собрать словарь `work_mode`, где значение `"удалённо"` для `remote == True`, иначе `"в офисе"`, но только если `hours ≥ 20`.
- **Критерии:** единственный dict comprehension, фильтр (`hours ≥ 20`) в конце выражения, тернарный оператор внутри значения, исходный список не изменяется.

## Задача 4. Подготовка к следующему уроку. «Логи авторизаций»
- **Блок:** set comprehension.
- **Описание:** список строк `logs = ["alex:login", "bob:update", "claire:logout"]`. Сформируй множество пользователей, у которых действие `login` или `update`, игнорируя строки, где формат отличается от `user:action`.
- **Критерии:** один set comprehension, проверка числа разделителей перед добавлением, строки приводятся к нижнему регистру только внутри comprehension при необходимости.
