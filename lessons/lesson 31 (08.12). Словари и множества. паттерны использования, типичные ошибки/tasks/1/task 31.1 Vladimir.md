# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Ремесленная мастерская помечает, какая станция обслуживания последний раз проходила диагностику. Мастер решает, какое оборудование ставить на проверку в первую очередь. Создай словарь diagnostics, где ключ — имя станции, а значение — дату диагностики. Затем выведи информацию для переменных check_tool и unused_tool.

## Пример входных данных
```python
stations = [
    ('laser', '2025-12-02 12:00'),
    ('cnc', '2025-12-04 09:45'),
    ('printer', '2025-12-05 15:20'),
]
check_tool = 'cnc'
unused_tool = 'lathe'
```

## Пример выходных данных
```text
cnc -> 2025-12-04 09:45
lathe -> не проверялась
```

## Критерии успешного выполнения
- Список пар преобразован в словарь diagnostics с ключом 'имя станции'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для check_tool и unused_tool в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку stations в цикле и наполняй словарь diagnostics.
- Используй множественное присваивание: key, value = запись из stations.
- Для отсутствующих данных возвращай строку 'не проверялась'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
