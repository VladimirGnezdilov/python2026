# Практическое задание 35.3

Это задание относится к третьему блоку урока, посвященному `*args`, `**kwargs` и распаковке. В предыдущих блоках вы научились работать с параметрами функций, позиционными и именованными аргументами, а также значениями по умолчанию. Сейчас вы переходите к созданию гибких функций, которые могут обрабатывать переменное количество входных данных — ключевой навык для написания универсального кода. Следующий блок будет посвящен области видимости переменных.

## Задания

### Описание задачи
Владимир, вы работаете над модулем для обработки данных о студентах. Вам нужно создать функцию `merge_student_data`, которая объединяет информацию о студенте из разных источников. Каждый источник предоставляет данные в виде именованных аргументов (например, `name`, `age`, `grades`). Однако количество и названия полей могут различаться. Ваша функция должна:
1.  Принимать обязательный позиционный аргумент `student_id` (идентификатор студента).
2.  Принимать произвольное количество дополнительных именованных аргументов с данными.
3.  Возвращать словарь, где ключом является `student_id`, а значением — еще один словарь, содержащий все переданные именованные аргументы.
4.  Также реализовать вспомогательную функцию `print_student_summary`, которая принимает словарь студента (результат `merge_student_data`) и выводит краткую информацию. Эта функция должна уметь распаковывать словарь при вызове.

**Контекст:** Это типичная задача при интеграции данных из нескольких систем (например, учебный портал, система оплат, библиотека), где каждый модуль добавляет свою информацию об одном и том же студенте.

### Пример входных и выходных данных

**Пример 1: Базовый вызов**
```python
# Вызов функции
student_1 = merge_student_data(101, name="Анна", age=20, faculty="ФИИТ")
print(student_1)

# Ожидаемый вывод:
# {101: {'name': 'Анна', 'age': 20, 'faculty': 'ФИИТ'}}
```

**Пример 2: Больше данных и использование распаковки**
```python
# Данные из одного источника
basic_info = {"name": "Игорь", "age": 21}
# Данные из другого источника
academic_info = {"grades": [5, 4, 5], "scholarship": True}

# Объединение с распаковкой словарей
student_2 = merge_student_data(102, **basic_info, **academic_info)
print(student_2)

# Ожидаемый вывод:
# {102: {'name': 'Игорь', 'age': 21, 'grades': [5, 4, 5], 'scholarship': True}}

# Использование вспомогательной функции с распаковкой
def print_student_summary(student_id, name, age, **kwargs):
    print(f"Студент {name} (ID: {student_id}), возраст: {age}. Доп. полей: {len(kwargs)}")

# Правильный вызов с распаковкой словаря student_2[102]
print_student_summary(**student_2[102])
# Ожидаемый вывод в консоль:
# Студент Игорь (ID: 102), возраст: 21. Доп. полей: 2
```

### Критерии проверки и ограничения
1.  Функция `merge_student_data` должна быть объявлена с параметрами `student_id` и `**kwargs`.
2.  Внутри функции `**kwargs` должен быть преобразован в словарь и помещен в результирующую структуру.
3.  Запрещается использовать циклы для обработки `kwargs` — используйте возможности словаря напрямую.
4.  Функция `print_student_summary` должна принимать конкретные параметры (`student_id`, `name`, `age`) и `**kwargs` для всего остального. При ее вызове для печати необходимо использовать распаковку (`**`) словаря с данными студента.
5.  Не используйте темы из будущих блоков: глобальные переменные, область видимости (`global`), замыкания.

### Решение задачи (псевдокод)
1.  Объяви функцию `merge_student_data` с параметрами `student_id` и `**kwargs`.
2.  Внутри функции создай пустой словарь `result`.
3.  Помести в `result` пару ключ-значение: ключ — `student_id`, значение — словарь `kwargs`.
4.  Верни словарь `result` из функции.
5.  Объяви функцию `print_student_summary` с параметрами `student_id`, `name`, `age` и `**kwargs`.
6.  Внутри этой функции напечатай строку, используя переданные `student_id`, `name`, `age`, и укажи количество дополнительных полей, вычислив длину словаря `kwargs`.
7.  В основном коде:
    *   Вызови `merge_student_data`, передав `student_id` и различные именованные аргументы.
    *   Для вызова `print_student_summary` получи словарь с данными студента по ключу `student_id` и распакуй его с помощью `**`.

### Рекомендации
-   Вынести в отдельную функцию: Логику объединения данных (`merge_student_data`) и логику вывода (`print_student_summary`) правильно вынесены в отдельные функции, что соответствует принципу единственной ответственности.
-   PEP8:
    -   Используйте осмысленные имена для параметров `**kwargs`. В документации к функции можно указать, что именно ожидается в `kwargs` (например, `**student_attributes`).
    -   Следите за длиной строки. Если вызов функции с множеством именованных аргументов становится слишком длинным, перенесите аргументы на новую строку с отступом.
