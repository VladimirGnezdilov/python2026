# Практическое задание 35.1

Это задание относится к первому блоку темы "Функции". Вы уже знакомы с коллекциями, алгоритмами и сортировками. Сейчас мы начинаем изучать функции — основной инструмент для структурирования кода. Этот блок закладывает фундамент: вы научитесь объявлять функции, передавать в них данные и получать результат. Следующие блоки будут строиться на этом понимании, добавляя значения по умолчанию, именованные аргументы и другие возможности.

## Задания

### Описание задачи
Владимир, вы работаете над системой учета простых геометрических расчетов для строительного проекта. Ваша задача — создать набор базовых функций для вычисления периметров и площадей прямоугольных участков.

Напишите три функции:
1.  `perimeter_rectangle` — принимает два аргумента (длину и ширину прямоугольника) и возвращает его периметр.
2.  `area_rectangle` — принимает два аргумента (длину и ширину прямоугольника) и возвращает его площадь.
3.  `compare_sizes` — принимает четыре аргумента (длину и ширину первого прямоугольника, затем длину и ширину второго). Функция должна вычислить площади обоих прямоугольников, сравнить их и вернуть строку:
    *   `"Первый больше"`, если площадь первого больше.
    *   `"Второй больше"`, если площадь второго больше.
    *   `"Равны"`, если площади равны.

**Важно:** Все функции должны использовать оператор `return` для передачи результата. Не используйте `print` внутри функций для вывода результата вычислений.

### Пример входных и выходных данных
Пример 1: Работа функций по отдельности
```python
# Вызов функций
p = perimeter_rectangle(5, 3)
a = area_rectangle(5, 3)
result = compare_sizes(5, 3, 4, 4)

print(p) # Должно вывести: 16
print(a) # Должно вывести: 15
print(result) # Должно вывести: "Первый больше"
```
*Пояснение:* Периметр = 2*(5+3)=16. Площадь = 5*3=15. Площадь первого прямоугольника (15) больше площади второго (4*4=16? Ошибка в примере? Проверим: 4*4=16, 15 < 16, значит второй больше. В примере опечатка, правильный вывод для compare_sizes(5,3,4,4) — `"Второй больше"`. Давайте исправим пример.

Исправленный пример:
```python
result = compare_sizes(5, 3, 4, 4)
print(result) # Должно вывести: "Второй больше" (т.к. 15 < 16)
```

Пример 2: Использование результата одной функции в другой
```python
# Можно использовать результат area_rectangle внутри compare_sizes "вручную",
# но сама функция compare_sizes должна делать вычисления сама.
a1 = area_rectangle(10, 2)
a2 = area_rectangle(5, 5)
print(a1) # 20
print(a2) # 25

# Сравнение через функцию
comparison = compare_sizes(10, 2, 5, 5)
print(comparison) # "Второй больше"
```

### Критерии проверки и ограничения
1.  Функции должны быть объявлены с помощью `def`.
2.  Функции `perimeter_rectangle` и `area_rectangle` должны иметь ровно два параметра.
3.  Функция `compare_sizes` должна иметь ровно четыре параметра.
4.  Все аргументы передаются в функции позиционно.
5.  Каждая функция должна заканчиваться оператором `return`, который возвращает вычисленное значение (число или строку).
6.  Не используйте в этом задании: значения по умолчанию, `*args`, `**kwargs`, именованные аргументы при вызове, глобальные переменные внутри функций.
7.  Внутри функций `perimeter_rectangle` и `area_rectangle` не должно быть вызовов `print`.

### Решение задачи (псевдокод)
1.  Объяви функцию `perimeter_rectangle` с двумя параметрами (назови их, например, `length` и `width`).
    *   Внутри функции вычисли значение: (length + width) * 2.
    *   Верни это значение с помощью `return`.
2.  Объяви функцию `area_rectangle` с двумя параметрами (например, `a` и `b`).
    *   Внутри функции вычисли значение: a * b.
    *   Верни это значение.
3.  Объяви функцию `compare_sizes` с четырьмя параметрами (например, `l1`, `w1`, `l2`, `w2`).
    *   Внутри функции вычисли площадь первого прямоугольника, используя `l1` и `w1` (можно мысленно вызвать логику `area_rectangle`, но не саму функцию, если это не требуется заданием. Для простоты вычисляем прямо здесь: `area1 = l1 * w1`).
    *   Вычисли площадь второго прямоугольника: `area2 = l2 * w2`.
    *   Сравни `area1` и `area2`:
        *   Если `area1` больше `area2`, верни строку `"Первый больше"`.
        *   Иначе, если `area2` больше `area1`, верни строку `"Второй больше"`.
        *   Иначе (значения равны), верни строку `"Равны"`.

### Рекомендации

*   PEP8:
    *   Используйте осмысленные имена для параметров, например `length`, `width`, `side_a`, `side_b`. Избегайте однобуквенных имен, кроме общепринятых `i`, `j` в циклах.
    *   После объявления функции `def...:` и перед кодом внутри функции оставляйте одну пустую строку для лучшей читаемости.
