# Задание 29.4. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Вложенные структуры. Практика чтения и фильтрации списков словарей для учета обращений клиентов.

## Условие задачи
Ты руководишь сервисной службой и собираешь отчёт по заявкам:
1. Создай список словарей `tickets` с ключами `id`, `client`, `devices` (список словарей `type`, `issue`), `priority`.
2. Выведи каждую заявку строкой `ID #. client — priority`, затем перечисли устройства и описания проблем.
3. Сформируй список всех устройств, для которых `issue` содержит слово из переменной `keyword`.
4. Построй словарь `client -> количество устройств`, подсчитав длину вложенных списков.
5. Отсортируй заявки по приоритету (например, `high > medium > low`) и выведи порядок их обработки.

## Пример входных данных
```python
tickets = [
    {
        "id": 101,
        "client": "Антон",
        "priority": "high",
        "devices": [
            {"type": "ноутбук", "issue": "не включается"},
            {"type": "планшет", "issue": "мигает экран"}
        ]
    },
    {
        "id": 102,
        "client": "Мария",
        "priority": "medium",
        "devices": [
            {"type": "смартфон", "issue": "сломана камера"}
        ]
    }
]
keyword = "экран"
```

## Пример выходных данных
```
ID 101. Антон. приоритет high
  - ноутбук. не включается
  - планшет. мигает экран
ID 102. Мария. приоритет medium
  - смартфон. сломана камера
Найденные устройства по ключу экран. [{'type': 'планшет', 'issue': 'мигает экран'}]
Клиенты и устройства. {'Антон': 2, 'Мария': 1}
Очередь по приоритету. [101, 102]
```

## Критерии успешного выполнения
- Данные заявок оформлены списком словарей с вложенными списками устройств
- Вывод формируется циклом с аккуратным обходом вложенных структур
- Фильтр по слову не изменяет исходные данные
- Словарь клиент -> количество устройств считается программно
- Сортировка по приоритету реализована через собственный порядок

## Подсказки
- Заведи словарь соответствия приоритетов: `{'high': 3, 'medium': 2, 'low': 1}`
- Для фильтра по слову используй сравнение `if keyword.lower() in issue.lower()`
- Чтобы распечатать устройства, пройдись вложенным циклом `for device in ticket['devices']`

## Частые ошибки и ограничения
- Не забывай, что `devices` уже список. не пытайся обращаться по ключам к строке
- Не изменяй исходный список при сортировке. работай с копией или списком `sorted`
- Следи за единым форматом вывода. каждая заявка печатается одинаково
- Не смешивай типы данных в словаре приоритетов
