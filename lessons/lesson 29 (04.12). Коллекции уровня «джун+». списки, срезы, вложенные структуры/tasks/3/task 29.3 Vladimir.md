# Задание 29.3. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Мутация и методы списков. Отрабатываем добавление, вставку и удаление элементов, чтобы гибко управлять текущим списком дел.

## Условие задачи
Ты ведешь очередь обслуживания в небольшом сервисном центре:
1. Создай список клиентов в порядке поступления заявок.
2. Добавь через `extend` заявки из онлайн формы, которые пришли позже.
3. Вставь «диагностика без записи» в позицию №3, чтобы обозначить срочный визит.
4. Удали по значению клиента, который отменил заявку.
5. Перемести предпоследний элемент в начало списка, используя `pop(-2)` и `insert(0, ...)`.
6. Выведи итоговый порядок с указанием типа заявки у каждого клиента.

## Пример входных данных
```python
queue = ["Антон, экран", "Марина, батарея", "Игорь, звук", "Вера, камера"]
late_requests = ["Павел, кнопки", "Даша, корпус"]
```

## Пример выходных данных
```
1. Игорь, звук
2. Антон, экран
3. Марина, батарея
4. диагностика без записи
5. Вера, камера
6. Даша, корпус
```

## Критерии успешного выполнения
- Использованы минимум три разных метода изменения списка
- Вставка, удаление и перемещение выполняются корректно
- Итоговый список распечатан после всех операций
- В выводе объясняется назначение каждого элемента

## Подсказки
- После `extend` новые заявки автоматически появятся в конце `queue`
- Индекс 2 вставит срочный визит третьим по порядку
- Перед `remove` убедись, что нужный клиент присутствует
- Для вывода с пояснениями удобно использовать `enumerate(queue, start=1)`

## Частые ошибки и ограничения
- Не меняй список в том же цикле, где его перебираешь
- Не вызывай `remove`, если не проверил наличие элемента
- Следи, чтобы индексы `pop` и `insert` не выходили за длину
- Работай только с исходным списком, не создавай копию
