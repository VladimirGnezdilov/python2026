# Вопрос 36. Тема 6: Основы асинхронного программирования

2) Только одно ядро
3) Всегда быстрее
4) Нельзя тестировать

### ИИ анализ
Асинхронность означает, что программа может приостановить выполнение одной задачи, если она ожидает (например, ввода-вывода), и переключиться на другую, не блокируя весь поток выполнения.  Это повышает общую эффективность, особенно в задачах с большим количеством операций ожидания, но не гарантирует, что код всегда будет выполняться быстрее.

# Вопрос 38. Тема 6: Основы асинхронного программирования

### ИИ анализ

# Вопрос 39. Тема 6: Основы асинхронного программирования

Что делает await?

Варианты ответов:
1) Создает список
2) Импортирует модуль
3) ✅ Ожидает завершения асинхронной операции
4) Печатает результат
5) Завершает программу

### ИИ анализ
Ключевое слово `await` приостанавливает выполнение текущей корутины до тех пор, пока не завершится асинхронная операция (например, сетевой запрос), которую оно ожидает.  В это время управление передается другим корутинам, что позволяет эффективно использовать время.  `await` можно использовать только внутри асинхронной функции (`async def`).

# Вопрос 40. Тема 6: Основы асинхронного программирования

# Вопрос 42. Тема 6: Основы асинхронного программирования

1) Всегда процессы
2) Всегда потоки
3) Пишут в один файл
4) ✅ Задачи чередуются в ожидании

Это отличается от параллельного выполнения, где задачи выполняются одновременно на разных ядрах процессора.
