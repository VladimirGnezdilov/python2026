## Задача 1. Система скидок (наследование + абстрактный класс + super)

Нужно реализовать простую систему расчёта скидок для интернет-магазина.

### Требуется

1. Создай абстрактный класс `DiscountPolicy`:
   - в `__init__` хранит `name` (например, `"Base"`)
   - имеет абстрактный метод `apply(price: float) -> float` (возвращает итоговую цену после скидки)
   - имеет обычный метод `clamp_price(price: float) -> float`, который:
     - если `price < 0`, возвращает `0`
     - иначе возвращает `price`

2. Создай класс `PercentDiscount(DiscountPolicy)`:
   - в `__init__` принимает `name` и `percent` (например, `10`)
   - переопредели `apply(price)` так, чтобы:
     - сначала нормализовать цену через `super().clamp_price(price)`
     - затем вернуть цену после процентной скидки

3. Создай класс `FixedDiscount(DiscountPolicy)`:
   - в `__init__` принимает `name` и `amount` (например, `250`)
   - переопредели `apply(price)` так, чтобы:
     - сначала нормализовать цену через `super().clamp_price(price)`
     - затем вернуть цену после вычитания фиксированной скидки
     - итоговая цена не должна быть меньше `0`

4. Создай класс `PriceCalculator`:
   - в `__init__` принимает список политик `policies: list[DiscountPolicy]`
   - метод `calculate(price: float) -> float` применяет политики по очереди и возвращает итоговую цену

5. В конце программы:
   - создай минимум 2 политики скидок (процентную и фиксированную)
   - передай их в `PriceCalculator`
   - посчитай цену минимум для 2 разных исходных цен

### Критерии готовности

- Нельзя создавать экземпляр `DiscountPolicy` напрямую.
- В `apply(...)` у наследников обязательно используется `super()`.
- `PriceCalculator` работает через тип `DiscountPolicy`, а не через конкретные классы.

## Задача 2. Новая политика скидки (наследование + super)

Расширь решение из задачи 1.

### Требуется

1. Создай класс `MinPriceDiscount(DiscountPolicy)`:
   - в `__init__` принимает `name`, `min_price` и `percent`
   - `apply(price)`:
     - сначала нормализует цену через `super().clamp_price(price)`
     - если цена меньше `min_price`, скидка не применяется (верни цену без изменений)
     - иначе верни цену после процентной скидки

2. Добавь `MinPriceDiscount` в `PriceCalculator` и проверь работу на ценах ниже и выше порога.
