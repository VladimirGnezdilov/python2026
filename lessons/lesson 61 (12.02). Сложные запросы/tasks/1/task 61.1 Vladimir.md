# Уровень 1 · Базовый

**Задача.** Создать структуры `Customers` и `Orders`, заполнить демонстрационными данными и получить перечень клиентов вместе с их заказами.

**Детали.**
- Схемы и типы данных совместимы с SQLite.
- Для примеров достаточно пары клиентов и заказов.
- Итоговый список должен включать имя, фамилию, номер заказа и сумму.

**Псевдокод решения.**
```
[описать структуру Customers]
    поле CustomerID как целочисленный идентификатор с автоувеличением
    поле FirstName как текст
    поле LastName как текст
    поле Email как текст

[описать структуру Orders]
    поле OrderID как целочисленный идентификатор с автоувеличением
    поле CustomerID как целочисленная ссылка на Customers.CustomerID
    поле OrderDate как текст в формате YYYY-MM-DD
    поле TotalAmount как число с плавающей точкой
    связь внешнего ключа между Orders.CustomerID и Customers.CustomerID

[добавить записи в Customers]
    запись 1: (1, "John", "Doe", "johndoe@example.com")
    запись 2: (2, "Jane", "Smith", "janesmith@example.com")

[добавить записи в Orders]
    запись 1: (101, 1, "2025-02-01", 100.50)
    запись 2: (102, 2, "2025-02-02", 200.75)

[сформировать выборку]
    источник A: Customers
    источник B: Orders
    условие соответствия: A.CustomerID равен B.CustomerID
    выбрать поля: A.FirstName, A.LastName, B.OrderID, B.TotalAmount
```

# Уровень 2 · Средний

**Задача.** Показать клиентов даже без заказов и сформировать единый список клиентов и заказов.

**Детали.**
- Использовать ранее созданные таблицы.
- Первый запрос должен включать всех клиентов, даже если заказов нет.
- Второй результат — единый перечень с объединением строк из обеих таблиц.

**Псевдокод решения.**
```
[запрос A]
    источник основной: Customers
    источник связанный: Orders
    условие соответствия: Customers.CustomerID равен Orders.CustomerID
    тип соединения: брать всех из Customers, сопоставлять при наличии заказов
    выбрать поля: Customers.FirstName, Customers.LastName, Orders.OrderID, Orders.TotalAmount

[запрос B]
    часть 1: выбрать Customers.FirstName, Customers.LastName, пустой OrderID, пустую TotalAmount
    часть 2: выбрать пустые FirstName и LastName, Orders.OrderID, Orders.TotalAmount
    объединить результаты частей без дубликатов
```

# Уровень 3 · Продвинутый

**Задача.** Найти заказы, для которых нет соответствующего клиента.

**Детали.**
- Использовать сравнение наборов данных.
- Результат должен показывать строки из `Orders`, которые отсутствуют после сопоставления с `Customers`.

**Псевдокод решения.**
```
[набор базовый]
    взять OrderID, CustomerID, OrderDate, TotalAmount из Orders

[набор сопоставленный]
    объединить Orders с Customers по CustomerID
    выбрать те же поля, что и в базовом наборе

[результат]
    вывести элементы базового набора, которых нет в сопоставленном наборе
```
