## Задание: `summarize_orders`
Реализуй функцию `summarize_orders(*orders, **kwargs)`, которая принимает кортежи вида `("товар", количество)` и необязательные параметры:
- `currency="RUB"` — строка с обозначением валюты;
- `round_to=2` — сколько знаков оставить у среднего значения;
- любые другие ключи могут приходить, но их нужно игнорировать, не копируя в результат.

Функция должна:
1. Проверять, что передан хотя бы один заказ. Если `orders` пустой, выбрасывать `ValueError("Нет заказов для обработки")`.
2. Сложить общее количество единиц товара.
3. Посчитать среднее количество единиц на заказ и округлить до `round_to`.
4. Вернуть словарь вида:
   ```python
   {
       "orders": <количество заказов>,
       "total_items": <общее количество>,
       "avg_items": <среднее количество>,
       "currency": <значение currency>
   }
   ```
5. Показывать в коде распаковку `*args`/`**kwargs` и аккуратную фильтрацию лишних ключей.

### Примеры сценариев
1. **Базовый**
   ```python
   summarize_orders(("кофе", 2), ("чай", 1))
   # {'orders': 2, 'total_items': 3, 'avg_items': 1.5, 'currency': 'RUB'}
   ```
2. **Изменённое округление и валюта**
   ```python
   summarize_orders(("пицца", 3), ("салат", 1), round_to=1, currency="USD")
   # {'orders': 2, 'total_items': 4, 'avg_items': 2.0, 'currency': 'USD'}
   ```
3. **Негативный тест**
   ```python
   summarize_orders()
   # ValueError: Нет заказов для обработки
   ```

### Подсказки
- Используй цикл для суммирования количеств.
- Проверку пустых `*args` делай в самом начале, чтобы не дублировать логику.
- Чтобы игнорировать лишние ключи, занеси в результат только необходимые значения из `kwargs`.

### Псевдокод решения
```
1. Объявляем summarize_orders, принимающую все заказы через *orders и параметры через **kwargs.
2. Сразу проверяем, что в *orders есть хотя бы один элемент, иначе выбрасываем ValueError с текстом о пустом списке.
3. Читаем валюту из kwargs по ключу "currency", при отсутствии используем значение "RUB".
4. Читаем precision из kwargs по ключу "round_to", при отсутствии используем число 2.
5. Создаём счётчик total_items и ставим в ноль.
6. Для каждого заказа распаковываем название и количество, добавляем количество к total_items.
7. Считаем avg_items как total_items, делённое на количество заказов, результат округляем до round_to.
8. Формируем словарь только с ключами orders, total_items, avg_items и currency и возвращаем его.
```
